# 金融宏观页重构设计

## 背景

当前金融宏观页的问题不是缺内容，而是信息架构失焦：右侧栏目过长、左侧内容短，用户需要不断下滑才能拼出结论。页面把数据、新闻、解释、日历、来源状态平铺出来，但没有先回答用户每天早上最关心的问题：当前宏观环境到底偏进攻、偏防守，还是观望。

用户确认的新方向是 **D 为主、B 为辅**：

- D：先给总判断，再能下钻原因。
- B：保留近期重要事件/日历能力，但不能让日历成为页面主角。

## 设计目标

新版页面应像一个每日早晨使用的宏观决策 cockpit：

1. 10 秒内看到总判断。
2. 30 秒内知道判断来自哪些证据。
3. 2 分钟内能查清近期哪些数据和新闻可能改变判断。
4. 默认不堆原始新闻、指标解释、数据源状态。
5. 每条展示信息都必须回答“所以呢”：它影响哪个资产、支持还是反驳当前判断、为什么重要。

## 非目标

- 不做完整投研报告。
- 不把所有新闻和所有指标默认铺开。
- 不用大段解释教育用户“为什么放进日历”。
- 不追求交易指令，只输出宏观环境和观察优先级。

## 页面结构

### 第一屏：总判断区

顶部使用深色判断块，展示：

- 总判断：例如“偏防守，但不是极端避险”。
- 一句话解释：例如“实际利率仍高、美元偏强，黄金相对风险资产更有解释力；BTC 暂不追，等待 ETH/BTC 或资金流确认。”
- 资产动作标签：黄金、BTC、美股、现金等，用“观察偏多 / 不追 / 等数据 / 保留机动”这类用户能直接理解的语言。

这一块必须是页面视觉重心，不被日历或新闻抢走。

### 第二层：四个证据卡

紧跟总判断下方，放四个紧凑证据卡：

- 潮水：利率、美元、实际利率、通胀和就业共同判断流动性松紧。
- 避险：黄金、原油、VIX、地缘/关税新闻判断避险需求。
- 币圈：BTC/ETH、BTC 市占率、币圈情绪判断风险偏好是否扩散。
- 情绪：恐慌贪婪、VIX、新闻冲击判断是否极端。

每张卡只显示结论和一条核心证据。详细解释放到下钻抽屉。

### 第三层：支持证据与反向证据

用两栏结构解释“为什么是这个结论”：

- 支持当前判断的证据。
- 反驳或削弱当前判断的证据。

这样用户不会只看到单边叙事，也能理解为什么不是“极端看多/看空”。

### 右侧：今日 / 本周必须盯

右侧不再做长栏。只保留：

- 今日 / 本周必须盯：3-5 条最高优先级事件。
- 资产温度：黄金、DXY、VIX、BTC 等核心资产快照。
- 下钻入口提示。

事件按重要度排序，而不是按数据源或抓取顺序排序：

- 高：可能改变总判断的事件，例如美国 PCE、非农、CPI、FOMC。
- 中：可能验证或削弱判断的事件，例如美联储讲话、OPEC、中东冲突。
- 低：普通新闻，仅在能解释资产变化时展示。

### 下方：完整风险日程

完整日历保留，但不作为第一屏主角。默认改成“7 天风险日程”，展示近期最重要的数据和新闻。

提供切换：

- 未来 7 天重点。
- 本月日历。
- 只看数据。
- 只看新闻。

新闻点击后打开抽屉，不再弹出冗长说明。

## 信息展示规则

### 新闻

新闻默认不展示全文，只展示：

- 标题。
- 来源和时间。
- 影响资产。
- 支持 / 反驳当前判断。
- 为什么重要。

需要删除类似“新闻按发布时间落到月历里，用来解释宏观倾向”这种从系统视角出发的说明。用户不需要知道系统为什么这么组织，只需要知道新闻对判断有什么用。

### 指标解释

指标解释不再作为右侧长列表默认展示。

放置方式：

- 点击数据事件时，在详情抽屉里显示“这个指标是什么 / 怎么看 / 对资产的含义”。
- 页面下方可以保留折叠的“指标解释库”，但默认关闭。

### 数据源状态

数据源状态默认隐藏到“系统状态”折叠区。只有失败时在页面底部或角标提醒，不占用核心页面空间。

## 数据结构需求

现有 `data/macro-latest.json` 基本可继续使用，但页面渲染层需要派生新结构：

- `headlineDecision`：总判断、解释、资产动作标签。
- `evidenceCards`：四个证据卡。
- `supportingEvidence`：支持判断的证据列表。
- `counterEvidence`：反向证据列表。
- `watchlist`：今日/本周必须盯的事件。
- `assetTemperature`：核心资产温度。

这些可以先在前端从现有 JSON 推导；后续再考虑由生成脚本直接写入。

## UI 原则

- 第一屏不能出现长滚动右栏。
- 卡片高度要可控，默认显示两到三行，超出折叠。
- 重点内容用结论性语言，不用数据源语言。
- 日历只是风险日程工具，不是主叙事。
- 原始数据、解释、新闻全文全部下钻。
- 页面应更像金融工作台，而不是资料库。

## 验证标准

实现完成后需要验证：

1. 桌面第一屏能同时看到总判断、四个证据卡、今日/本周必须盯、资产温度。
2. 右侧不出现明显长于左侧的无限列表。
3. 页面默认不展示完整指标解释库。
4. 新闻详情不再出现系统自述式文案。
5. 在移动端，内容顺序为：总判断、资产动作、证据卡、必须盯、风险日程。
6. `macro.html` JS 语法检查通过。
7. GitHub Pages 页面可访问。

## 实施范围

主要修改：

- `macro.html`：重构布局、交互和展示文案。

可能修改：

- `scripts/macro-akshare.py`：如果前端推导结论不够清晰，再把 headline/watchlist 等结构写入 JSON。
- `data/macro-latest.json`：运行一次宏观脚本刷新数据。

不修改：

- 热点页 `index.html`。
- 热点自动化脚本。
